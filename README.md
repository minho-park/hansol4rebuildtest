# 이름+휴대폰 인증 영상 페이지

허용된 (이름, 휴대폰) 조합을 입력한 사용자에게만 유튜브 영상을 보여주는
정적 웹페이지. GitHub Pages 같은 정적 호스팅에 그대로 올려서 사용한다.

## 동작 방식

1. 사용자가 이름과 휴대폰 번호를 입력한다.
2. 브라우저에서 PBKDF2로 `이름 + 휴대폰` → AES-GCM 키를 도출한다.
3. `data.json`에 들어 있는 암호문 블롭들을 그 키로 차례대로 복호화 시도한다.
4. 하나라도 11자 형식의 유튜브 영상 ID로 풀리면 성공 → 영상 임베드.
5. 어느 것도 풀리지 않으면 실패.

영상 ID 자체가 암호화되어 있어서, 페이지 소스/`data.json`만 봐서는
영상 URL을 알 수 없다. 이름이 더해져서 키스페이스가 넓어졌기 때문에
무차별 대입 비용도 휴대폰만 쓸 때보다 훨씬 비싸다.

### 입력 정규화 규칙

- 휴대폰: 숫자만 추출 (하이픈/공백 무시) → `010-1234-5678` = `01012345678`
- 이름: 앞뒤 공백 제거 + 유니코드 NFC 정규화. **그 외에는 그대로 비교**.
  대소문자도 구분하므로, 등록 시 사용한 표기와 정확히 같아야 한다.

## 사용 방법

### 1. 영상 준비

- 유튜브에 영상을 올리고 공개 범위는 **일부 공개(Unlisted)** 로 설정.
- 영상 페이지 URL에서 `v=` 뒤의 11자(예: `dQw4w9WgXcQ`)가 영상 ID.

### 2. data.json 만들기

브라우저에서 `admin.html`을 열고:

1. 유튜브 영상 ID 입력
2. 허용할 항목을 한 줄에 하나씩 `이름, 휴대폰` 형식으로 입력
   - 예: `홍길동, 010-1234-5678`
   - 휴대폰은 형식 자유(숫자만 추출), 이름은 등록 시 표기 그대로 입력해야 한다
3. **생성** 클릭 → 결과를 **다운로드** 또는 **복사**해서 `data.json`에 덮어쓰기

`admin.html`은 외부로 데이터를 전송하지 않는다(전부 브라우저 안에서 처리).

### 3. GitHub에 업로드

다음 파일만 올린다:

- `index.html`
- `app.js`
- `style.css`
- `data.json`

`admin.html`과 `README.md`는 올려도 보안상 문제는 없지만, 굳이 공개할 필요는
없다. 안 올리고 싶으면 `.gitignore`에 추가하거나 그냥 빼고 푸시하면 된다.

### 4. GitHub Pages 활성화

레포 Settings → Pages → Branch를 `main`(혹은 `master`)으로 설정.

## 보안 한계 — 꼭 읽어볼 것

이 방식은 **캐주얼한 무단 접근을 막을 뿐 강력한 보안이 아니다.** 정적
호스팅 + 클라이언트 사이드 인증의 구조적 한계 때문이다.

- **무차별 대입 가능성**: 휴대폰만 쓸 때 키스페이스가 ~10^8(1억)이었다면,
  이름+휴대폰은 현실적인 한국 이름 다양성을 곱해 ~10^13~10^14 정도가 된다.
  PBKDF2 25만 회와 합쳐 일반 공격자에겐 사실상 비현실적이지만, 표적이
  특정인이고 공격자가 그 사람의 이름을 안다면 키스페이스는 다시 휴대폰만큼
  좁아진다. 즉, **"이름은 비밀이 아닌 식별자"** 라는 점을 잊지 말 것.
- **영상 ID 노출**: 인증에 성공하면 브라우저 DevTools(네트워크/요소 검사)로
  영상 ID를 추출할 수 있다. 추출된 ID를 `youtube.com/watch?v=ID`에 붙여넣으면
  유튜브에서 직접 시청 가능 → 즉, **유튜브 영상 자체를 "내 페이지에서만"
  보게 만드는 것은 원리적으로 불가능**. 일부 공개(Unlisted)는 검색 노출만
  막을 뿐 URL 직접 접근은 허용한다.
- **클라이언트 사이드의 한계**: `data.json`과 JS는 누구나 다운받을 수 있다.

### 더 강한 보안이 필요하다면

- **유튜브 비공개(Private)** + 시청자의 Google 계정을 직접 추가 (전화번호
  인증 흐름과는 별개의 메커니즘)
- 서버를 두고 인증 후 **단기 서명 URL** 발급 (예: Cloudflare Stream,
  Mux 같은 비디오 호스팅 서비스)
- 영상 자체를 암호화해서 서빙하고 페이지에서만 키를 받아 디코딩

위는 GitHub Pages 정적 호스팅 범위를 벗어나는 작업이 필요하다.

## 파일 구조

```
.
├── index.html      사용자용 인증 페이지
├── app.js          인증/복호화 로직
├── style.css       스타일
├── data.json       암호화된 영상 데이터 (admin.html로 생성)
├── admin.html      data.json 생성기 (관리자용, 로컬 실행 권장)
├── README.md       이 문서
└── .gitignore
```

## 로컬 테스트

`fetch("data.json")`이 `file://`에서는 막히므로 로컬 서버를 띄워야 한다:

```sh
cd /path/to/EncryptionWebTest
python3 -m http.server 8000
# http://localhost:8000 접속
```
